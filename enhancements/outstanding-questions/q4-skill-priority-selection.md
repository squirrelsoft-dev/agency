# Question 4: Skill Priority Selection

## Context
Proposed 6 workflow-specific skills for the agency plugin. Need to prioritize which to create first.

## Question
Which 4-6 skills are most important to create for Phase 1?

## Proposed Skills

### 1. Agency Workflow Patterns ⭐
**Priority**: HIGH
**Rationale**: Core to understanding how to use the agency system effectively

**Content**:
- Orchestrator usage patterns
- Multi-agent coordination
- Parallel vs sequential execution
- Context management strategies
- Common workflow patterns

### 2. GitHub Workflow Best Practices ⭐
**Priority**: HIGH
**Rationale**: Most commands are GitHub-focused

**Content**:
- Branch naming conventions
- PR creation and review
- Issue triage strategies
- Sprint planning
- Git worktree usage
- Commit message standards

### 3. Code Review Standards ⭐
**Priority**: HIGH
**Rationale**: Critical for quality gates in workflows

**Content**:
- Security review checklist
- Code quality criteria
- Performance considerations
- Testing requirements
- Documentation expectations

### 4. Testing Strategy & Standards ⭐
**Priority**: HIGH
**Rationale**: Essential for `/agency:test` and verification phases

**Content**:
- Test pyramid
- Framework selection
- Coverage thresholds
- Test organization
- Mocking strategies
- CI/CD integration

### 5. Documentation Standards
**Priority**: MEDIUM
**Rationale**: Useful but not critical for core workflows

**Content**:
- Code comment guidelines
- ADR writing
- API documentation
- README structure

### 6. Development Workflow Automation
**Priority**: MEDIUM
**Rationale**: Supplements hooks, may be redundant

**Content**:
- Pre-commit hooks
- CI/CD patterns
- Automated code review
- Tool integration

## Recommendation
Create skills 1-4 in Phase 1 (all marked ⭐), defer 5-6 to Phase 2 or based on user feedback.

## Decision Needed
- Confirm priority ranking
- Should all 6 be created, or just top 4?
- Any additional skills to consider?
