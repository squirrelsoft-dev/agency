## Selection Algorithm

### Multi-Dimensional Matching

When selecting an agent for a task, evaluate across these dimensions:

```python
def score_agent_match(agent, task):
    score = 0.0

    # 1. Domain Match (30% weight)
    if task.domain in agent.domains:
        score += 30

    # 2. Technology Match (25% weight)
    tech_overlap = len(set(task.technologies) & set(agent.technologies))
    score += (tech_overlap / len(task.technologies)) * 25 if task.technologies else 0

    # 3. Complexity Match (20% weight)
    complexity_diff = abs(task.complexity - agent.complexity_rating)
    score += max(0, 20 - (complexity_diff * 5))

    # 4. Phase Match (15% weight)
    if task.phase in agent.phases:
        score += 15

    # 5. Tool Requirements (10% weight)
    tool_overlap = len(set(task.required_tools) & set(agent.available_tools))
    score += (tool_overlap / len(task.required_tools)) * 10 if task.required_tools else 10

    return score
```

## Usage Guidelines

### When to Use Single Agent
- Simple, focused tasks
- Clear domain boundaries
- Low complexity
- Single technology stack
- Quick fixes or updates

### When to Use Multiple Agents
- Multi-domain tasks (design + implementation)
- Medium complexity requiring specialized skills
- Need quality validation (dev + QA)
- Cross-functional features

### When to Use Orchestrator
- Complex multi-step workflows
- 3+ agents required
- Parallel execution opportunities
- Quality gates and retry logic needed
- Full project lifecycle (planning ‚Üí implementation ‚Üí testing)
